Complex Expressions

Full JavaScript expressions in templates with auto-detection

← Back to Examples

Usage

// Simple property access (auto-detected as safe, CSP-compatible)
<span>{this.state.count}</span>

// Complex expressions (auto-detected, uses new Function)
<span>{ this.state.count * 2 }</span>
<span>{ this.state.active ? 'Yes' : 'No' }</span>
<span>{ this.state.name.toUpperCase() }</span>

// In conditions
<veda-if condition="{ this.state.items.length > 0 }">
  ...
</veda-if>

// In loops
<veda-loop items="{ this.state.items.filter(x => x.active) }">
  ...
</veda-loop>

⚠️ Security Note

Complex expressions use new Function() internally for evaluation.

  • Safe: Hardcoded templates written by developers
  • Safe: Expressions that read user data: { this.state.userInput }
  • Dangerous: Templates from database/API
  • Dangerous: Dynamic expression construction with user input

See SECURITY.md for details.
