Virtual List

Efficient rendering of large lists (10,000+ items) using virtualization.

How Virtual Scrolling Works

  • DOM Efficiency: Only ~15-20 DOM elements exist regardless of list size
  • Memory Savings: 10,000 items use same memory as 20 items
  • Smooth Scrolling: Overscan renders extra items above/below viewport
  • Reactive: Adding/removing items automatically updates the view

Usage

<Virtual items="{this.state.items}" height="400" item-height="50">
  <Loop items="{this.visibleItems}" key="id" as="item">
    <div class="row">{item.name}</div>
  </Loop>
</Virtual>

Attributes

  • items - Source array expression
  • height - Viewport height in pixels (or "auto")
  • item-height - Height of each item in pixels
  • overscan - Extra items to render (default: 3)
